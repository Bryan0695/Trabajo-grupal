# Manejo de 3D-Secure (3DS) — PLUX

## ¿Qué es 3DS?

3D Secure es una capa adicional de seguridad para pagos online con tarjeta. El banco emisor determina si se requiere o no, dependiendo del análisis de riesgo de la transacción.

---

## Escenarios posibles con 3DS

### 1. FrictionLess (sin desafío)
El banco aprueba directamente sin solicitar validación adicional al cliente.
- **La respuesta es code 0 (pago exitoso)** directamente
- No se requiere segunda llamada a la API
- El flujo es idéntico al pago normal

### 2. Desafío (challenge)
El banco detecta una transacción que requiere verificación adicional.
- La API retorna `code: 103`
- Se debe redirigir al cliente a la pantalla de desafío del banco
- El banco envía un token al email/teléfono registrado del tarjetahabiente
- Hay dos sub-versiones: **3DS V1** y **3DS V2**

---

## Cómo identificar el tipo de 3DS

La respuesta con `code: 103` incluye el campo `customFormChallenge`:
- `customFormChallenge: false` → **3DS V2** (redirigir directamente a la URL)
- `customFormChallenge: true` → **3DS V1** (requiere llamada intermedia POST)

---

## Respuesta con code 103

```json
{
  "code": 103,
  "description": "Validación 3ds necesaria",
  "detail": {
    "url": "https://url-del-banco.com/desafio",
    "idTransaction": "Y2E1MzA3MjAtMjk5Yy0xMWVlLT...",
    "customFormChallenge": false,
    "parameters": []
  },
  "status": "succeded"
}
```

## Flujo 3DS V2 (customFormChallenge = false)

```
1. API responde code 103 con detail.url
2. Redirigir al navegador del cliente a detail.url (GET directo)
3. El banco muestra su formulario de desafío (token por SMS/email)
4. El cliente completa el desafío en la página del banco
5. El banco redirige al urlRetorno3ds enviado en la petición original
6. La URL de retorno recibe: ?pti=...&pcc=...&ptk=...
7. Hacer POST a integrations/dataTransactionThreeDsResource con esos parámetros
8. Recibir respuesta final del pago
```

**Código JavaScript para redirigir (3DS V2):**
```javascript
function manejar3DSV2(respuesta3DS) {
  if (!respuesta3DS.detail.customFormChallenge) {
    // 3DS V2: redirigir directamente a la URL
    window.location.href = respuesta3DS.detail.url;
  }
}
```

---

## Flujo 3DS V1 (customFormChallenge = true)

```
1. API responde code 103 con detail.url + detail.parameters
2. Crear un formulario HTML POST con la URL y los parámetros
3. Enviar el formulario (esto carga la página de desafío del banco)
4. El cliente completa el desafío
5. El banco redirige al urlRetorno3ds
6. Hacer POST a integrations/dataTransactionThreeDsResource
7. Recibir respuesta final del pago
```

**Código para 3DS V1 (jQuery):**
```html
<form id="form3DS" method="POST"></form>
<script>
function manejar3DSV1(detail) {
  const form = document.getElementById('form3DS');
  form.action = detail.url;
  
  detail.parameters.forEach(campo => {
    const input = document.createElement('input');
    input.type = 'hidden';
    input.name = campo.name;
    input.value = campo.value;
    form.appendChild(input);
  });
  
  form.submit();
}
</script>
```

**Código para 3DS V1 (JavaScript puro):**
```javascript
function manejar3DSV1(detail) {
  const form = document.createElement('form');
  form.method = 'POST';
  form.action = detail.url;
  
  detail.parameters.forEach(campo => {
    const input = document.createElement('input');
    input.type = 'hidden';
    input.name = campo.name;
    input.value = campo.value;
    form.appendChild(input);
  });
  
  document.body.appendChild(form);
  form.submit();
}
```

## Handler completo para code 103

```javascript
function manejar3DS(respuesta) {
  const detail = respuesta.detail;
  
  if (detail.customFormChallenge === false) {
    // 3DS V2: redirección directa
    window.location.href = detail.url;
  } else {
    // 3DS V1: formulario POST intermedio
    manejar3DSV1(detail);
  }
}
```

## Paso final: Confirmar el pago tras 3DS

Cuando el banco redirige al `urlRetorno3ds`, esa URL recibe los parámetros `pti`, `pcc`, `ptk`. Tu servicio debe hacer un POST para cerrar el pago:

### Endpoint de confirmación 3DS
```
POST {BASE_URL}integrations/dataTransactionThreeDsResource
Content-Type: application/json
Authorization: Basic {base64(id_cliente:clave_secreta)}
```

### Body
```json
{
  "pti": "valor_recibido_en_query_param",
  "pcc": "valor_recibido_en_query_param",
  "ptk": "valor_recibido_en_query_param"
}
```

| Campo | Oblig. | Descripción |
|---|---|---|
| `pti` | Sí | ID de la transacción (query param de la URL de retorno) |
| `pcc` | Sí | Código CVV de la tarjeta (query param) |
| `ptk` | Sí | Token de la transacción (query param) |

### Código Node.js para el servicio de retorno 3DS:
```javascript
// Este es un servicio GET que PLUX llama con los query params
app.get('/confirmar-pago', async (req, res) => {
  const { pti, pcc, ptk } = req.query;
  
  if (!pti || !pcc || !ptk) {
    return res.status(400).json({ error: 'Parámetros 3DS incompletos' });
  }
  
  try {
    const authBase64 = Buffer.from(`${ID_CLIENTE}:${CLAVE_SECRETA}`).toString('base64');
    
    const response = await axios.post(
      `${BASE_URL}integrations/dataTransactionThreeDsResource`,
      { pti, pcc, ptk },
      {
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Basic ${authBase64}`
        }
      }
    );
    
    const resultado = response.data;
    
    if (resultado.code === 0) {
      // Pago exitoso - actualizar orden en base de datos
      await actualizarOrden(resultado.detail.token, 'PAGADO');
      return res.redirect('/pago-exitoso?token=' + resultado.detail.token);
    } else {
      // Pago fallido
      await actualizarOrden(pti, 'FALLIDO');
      return res.redirect('/pago-fallido?razon=' + encodeURIComponent(resultado.description));
    }
  } catch (error) {
    console.error('Error confirmando pago 3DS:', error);
    return res.status(500).json({ error: 'Error procesando confirmación' });
  }
});
```

### Código Python (Flask) para el servicio de retorno 3DS:
```python
from flask import Flask, request, redirect
import requests
import base64

app = Flask(__name__)

@app.route('/confirmar-pago', methods=['GET'])
def confirmar_pago_3ds():
    pti = request.args.get('pti')
    pcc = request.args.get('pcc')
    ptk = request.args.get('ptk')
    
    if not all([pti, pcc, ptk]):
        return {'error': 'Parámetros 3DS incompletos'}, 400
    
    credenciales = f"{ID_CLIENTE}:{CLAVE_SECRETA}"
    auth = "Basic " + base64.b64encode(credenciales.encode()).decode()
    
    response = requests.post(
        f"{BASE_URL}integrations/dataTransactionThreeDsResource",
        json={"pti": pti, "pcc": pcc, "ptk": ptk},
        headers={
            "Content-Type": "application/json",
            "Authorization": auth
        }
    )
    
    resultado = response.json()
    
    if resultado['code'] == 0:
        token = resultado['detail']['token']
        return redirect(f'/pago-exitoso?token={token}')
    else:
        return redirect(f'/pago-fallido?razon={resultado["description"]}')
```

---

## Importante: URL de retorno sin urlRetorno3ds

Si no se envía `urlRetorno3ds` en la petición original, PLUX redirige a su propia pantalla de resultado. En este caso:
- **Se debe usar el Webhook** para recibir la notificación del resultado del pago
- El Webhook se configura en Dashboard → sección Developers → Web Hook
- No hay flujo en línea sin el webhook cuando no se usa `urlRetorno3ds`

```
     │                           │                           │
     │──── Redirige a url ──────────────────────────────────►│
     │                           │                           │
     │                           │    Desafío (OTP banco)    │
     │◄─────────────────────────────────────────────────────│
     │                           │                           │
     │──── POST /confirmar?pti=..&pcc=..&ptk=.. ────────────►│

```
